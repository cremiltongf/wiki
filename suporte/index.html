Tutoriais de suporte

Roteiro de alguns problemas mais comuns para rápida consulta e auxílio.

Usuário temporário

Com uma conta de Administrador faça os procedimentos a seguir:

  1. Abra o Regedit e navegue até:
    2. HKEY_LOCAL_MACHINE ->SOFTWARE->Microsoft->Windows NT->CurrentVersion->ProfileList
  2. Selecione as que tem o nome bem extenso. Por exemplo:
    3. S-1-5-21-3402367454-1173906501-2481948151-1226
  3. Encontre a pasta que pertence ao usuário problemático e exclua;
  4. Acesse o diretório de usuários:
    5. C:\Users
  5. Renomeie a pasta do usuário problemático. Por exemplo:
    6. atual: fulano virou renomeado: fulano.old
  6. Reinicie o Windows e crie uma nova conta para usuário. Copie os arquivos da pasta do usuário antigo para a nova.

Removendo vírus de Pen Drive

  1. Abra o prompt como Administrador e acesse a unidade do Pen Drive. Exemplo: f:
  2. Com o acesso a unidade do Pen Drive estabelecido, execute:
    3. attrib -r -a -s -h /s /d *.*
  3. Após o fim do processo, vamos excluir os atalhos:
    4. del *.lnk

Entenda o funcionamento de Attrib↗.

Gerenciando serviços em massa

Exemplo de uso:

auto: Automatico
disabled: Desativado
delayed-auto: Automatico com atraso na inicializacao

rem OCS Iventory = OCS Inventory Service
sc config "OCS Inventory Service" start= delayed-auto

Sub-process /usr/bin/dpkg returned an error code (1)

A solução abaixo se aplica ao Java.

sudo rm -r /var/lib/dpkg/info/oracle*
sudo rm -r /var/cache/apt/archives/oracle*
Geralmente ocorre por causa de interrupção na instalação de algum pacote. O mesmo procedimento se aplica a outros pacotes, quando o problema for semelhante.

Boot Repair

Usado para reparar o grub.

sudo add-apt-repository ppa:yannubuntu/boot-repair
sudo apt-get update
sudo apt-get install boot-repair

Se após o processo do Boot Repair o problema persistir, execute o comando abaixo no Windows 10 como Administrador:

bcdedit /set {bootmgr} path \EFI\ubuntu\grubx64.efi

Reinicie para testar.
